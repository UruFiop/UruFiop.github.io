Hugo + GitHub Pagesでブログ作成 - Hugoの基本を知る

11月 11, 2025
読了時間: 6分

自分のブログを作りたい！

自分のブログを作りたいと思った。
そう、エンジニア2年目に突入するにもかかわらず、自分のブログを持っていないことに気づいたのである。

使用する技術の選定

ブログを作るとしても、使う技術は様々。
少し考えただけでも以下の選択肢がある。

技術 感想
HTML + CSS ブログを作成するには大変か…？
ブログサービス(はてなブログなど) せっかくブログを作るなら、ちょっと技術的なことをやりたい
WordPress 環境作りやサーバーの確保を考えるとめんどくさい
React Reactをガリガリ書くのはめんどくさい
Next.js デファクトスタンダード感があるし、面白そう
Astro パフォーマンスが良く、しかも新しい
Hugo 比較的簡単にブログを作れそう

流石にHTML + CSSは論外だし、ブログサービスも面白みがなさそう。
WordPressも最初がめんどくさそうだし、セキュリティ対策が大変だと聞くしなぁ…

ReactもNext.jsもAstroもNode.js系で、Node.jsには色々とトラウマを抱えているので、今回は Hugo を使うことにした。

Hugoをインストールする

さっそく私の友達、GeminiさんにHugoの試し方を聞いてみよう。

  1. Hugoのインストール まず、お使いのOSにHugoをインストールする必要があります。Hugoの公式サイトには、各OS（Windows、macOS、Linux）ごとのインストール方法が記載されています。

まずはインストールが必要なようだ。
私はWinGet信者なので、WinGetを使用してインストールを行う。

サイトの作成

Geminiさんの回答の続きを見ていこう。

  1. 新しいサイトの作成 Hugoがインストールできたら、コマンドで新しいブログサイトの雛形を作成します。

Hugoではコマンドを一つ実行するだけで簡単にサイトを作れるようだ。

うーん、すごい簡単。

テーマの追加

  1. テーマの追加 Hugoサイトのデザインはテーマによって決定されます。
    ①テーマを探す: Hugo Themesのウェブサイトなどで気に入ったテーマを見つけます。

Hugoはテーマというものが存在するらしい。
WordPress民にとっては馴染みのある概念だと思うが、要はブログの外見を簡単に設定できるテンプレートのようなものだ。

テーマについては、以下のサイトから調べることができる。
Hugo Themes

その中から、自分が色々見てよいと思ったものをピックアップしてみた。

テーマ 感想
PaperMod とてもシンプル。技術系ブログに似合いそう
Stack モダンで見栄えが良い。一番気に入った
Blowfish ふぐがかわいい。デザイナーの人にいいかも
Hugo Profile 就活向け。ポートフォリオを紹介するのによさそう

各テーマは紹介ページのDemoボタンからデモページが見れるので、ぜひ活用されたし。

私はStackが一番気に入ったので、これを使用することにした。

②テーマをダウンロード: サイトのディレクトリ (my-blog) 内に、テーマをGitのサブモジュールとして追加するのが一般的です。

テーマのダウンロードも簡単で、先ほど作ったフォルダの中で以下のコマンドを実行するだけ。
なお、テーマの名前はGitHubの名前である。

1
2
3

C:\my-blog>git init

C:\my-blog>git submodule add [テーマのリポジトリURL] themes/[テーマの名前]

テーマのリポジトリURLについては、テーマの紹介ページのDownloadを押すとGitHubに飛ぶので、
緑色のCodeボタンを押した後にコピーボタンを押すとURLがコピーできる。

③テーマを設定: サイトの設定ファイル（通常は config.toml または hugo.toml）に、使用するテーマを指定します。

先ほど作ったフォルダの中にconfig.tomlがあると思うので、以下のように変更する。

1
2
3
4

baseURL = 'http://localhost:1313/' # 自分のサイトURL
languageCode = 'ja-jp' # デフォルトはen-usだが、日本に変更
title = "UruFiop's Blog"
theme = 'テーマの名前' # 追加したテーマの名前に変更

記事の作成

  1. コンテンツ（記事）の作成

コンテンツの作成には以下のコマンドを使う。

1

hugo new posts/my-first-post.md # 好きなファイル名にして良い(ex. saisyo.md)

すると、content/postsフォルダの中にmy-first-post.mdが作成されていると思う。
これで記事は作成できた。簡単。

作成したファイルを開いてみる。

1
2
3
4
5

+++
date = '2025-11-11T22:42:11+09:00'
draft = true # trueだと記事を公開しない、公開するときはfalseにする
title = 'My First Post'
+++

記事は+++の下にMarkdown形式で記載していく。
Markdown形式についてはこちらの記事を参照するとよい。

ローカルで確認

  1. ローカルでのプレビュー 記事を書き進める際は、ローカルサーバーを起動し、ブラウザでリアルタイムにサイトを確認できます。

以下のコマンドを使うと、ローカルで自分のブログを確認できる。

1

C:\my-blog>hugo server -D # -Dはドラフト記事(まだ公開していない記事)も確認できる引数

実行したら、http://localhost:1313/から自分のブログを確認できる。
もちろん、ローカルで確認しているだけなので、ネット上に公開されているわけではない。

サイトのデプロイ(公開用ファイルの作成)

  1. サイトの公開（デプロイ） コンテンツが完成したら、サイトを公開するための静的ファイルを生成します。 ①ドラフト設定を解除: 公開したいすべての記事のフロントマターで draft: true を draft: false に変更します。

記事ファイルの中のdraft = trueをdraft = falseにすると、その記事は公開状態になる。
自慢の記事が公開されないのは悲しいよね…

②静的ファイルを生成

以下のコマンドを実行するとpublicフォルダの中に公開用のファイルが作成される。

1

C:\my-blog>hugo

あとは、publicフォルダの中をGitHub PagesやNetlifyなどのホスティングサービスにアップロードすれば、自分のブログが公開できる。

まとめ

本記事ではHugoを使って、公開用ファイルの作成まで行った。
ただ、これだと日本語のフォントが不自然だったり、いまいち没個性的なサイトになってしまう。
